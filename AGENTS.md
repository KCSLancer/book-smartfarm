# AGENTS.md - 스마트팜 실패 경험 실용서 프로젝트

## 프로젝트 개요

10여년 회사생활을 한 저자가 인삼 스마트팜에 투자하여 실패한 경험을 바탕으로, 귀농/스마트팜을 준비하는 사람들을 위한 실용 가이드북을 집필하는 프로젝트.

- **책 제목**: 농사가 아닌 농업: 회사원이 마주한 스마트팜의 현실
- **장르**: 실용서/가이드북 (단행본)
- **대상 독자**: 귀농 준비자, 스마트팜 투자 고려자, 농업 현실을 알고 싶은 예비 농업인
- **톤**: 냉정하고 건조한 현실 직시. 감성적 미사여구, 동기부여 표현 없음
- **분량**: 약 236페이지 (14장, 64절, 6부 구성)
- **언어**: 한국어

## 글쓰기 원칙

### 목적과 방향

- **핵심 목적**: as-is(실패에 도달하게 된 배경)에 대한 고찰과 반성 → to-be(보완책)를 고민하는 내용
- **독자 가치**: 농업/스마트팜에 대해 잘 모르지만 이상적이고 희망찬 모습만을 가진 이들에게 직접적으로 도움이 될 정보 제공
- **톤의 원칙**: 정부/농협/기관을 비판하기보다 현실을 알려주고 판단에 도움. 판단은 독자의 몫

### 서사 구조: As-Is → To-Be

| 구분 | 해당 부분 | 역할 |
|------|-----------|------|
| Framing | 프롤로그 | 왜 이 기록을 남기는가 |
| As-Is (현실 분석) | 제1부~제4부 (1-12장) | 실패의 배경, 구조적 문제, 기술 한계, 개인 실패 원인, 시장 현실 |
| To-Be (보완책) | 제5부~제6부 (13-14장) | 다시 한다면의 개선안, 자기 점검 |
| Framing | 에필로그+부록 | 마무리, 체크리스트 |

### 사실검증 원칙

- **모든 정책/통계 주장은 정부 공식 자료로 검증** 후 본문에 반영
- **1차 자료원**: KOSIS 국가통계포털, 농림축산식품부, 농촌진흥청
- **2차 자료원**: 공공데이터포털, 한국농촌경제연구원, 국가법령정보센터
- **검증 가이드**: reference/fact_check_sources.md, reference/fact_check_quick_reference.md 참조
- **인용 형식**: 아래 "인용 및 데이터 원칙" 참조. 각주는 실용서에 부적합하므로 불사용

### IT 용어 순화 원칙 (4-5장 적용 완료, 본문 집필 시에도 적용)

- IoT → 센서 연결
- 클라우드 → 데이터 서버
- SaaS → 구독형 서비스
- 빅데이터 → 대량의 데이터
- DBA → 데이터 관리 전문가
- happy path → 이상적인 상황
- edge case → 예외 상황
- garbage in, garbage out → 잘못된 입력이면 잘못된 결과
- IoT 게이트웨이 → 센서 연결 장치
- 5장 이외에서 IT 전문 용어는 일상어로 풀어서 표현. "IT스러운 글이 아닌, 어려운 IT를 자연스럽게 풀어주는 글"

### 인용 및 데이터 원칙

- **출처 표시 적극적으로**: 본문은 흐름을 해치지 않도록 유지하고, 정책·통계·법령·시장 구조 관련 근거는 별도의 참고자료 페이지에 장별로 정리. 보고서식 서지 표기보다 독자가 웹검색으로 원문을 찾을 수 있도록 기관명과 찾아볼 자료를 자연스럽게 안내하는 방식 우선
- **데이터표 미제공**: 통계표나 수치표를 본문에 직접 옮기지 않음 (옮길 때 오류 가능성). 출처만 명시하여 독자가 직접 확인하도록 안내
- **정부 스마트팜 진단**: 참고용으로 기술. 정부 진단의 선택 기준(수치적 안정성)과 현장 경험에 의한 선택은 다를 수 있음을 명시

### 제외 확정 항목

- **가족 갈등**: 많이 겪는 일이지만 의견을 줄 수 없음. 반대하면 못하는 일
- **귀농 후 생활 환경 적응**: 의견을 줄 수 없음. 대부분 미리 고민하는 사안

### As-Is 분석 방법론 (Oracle 컨설팅 확정)

- **Gap Analysis** — 책 전체의 뼈대. "독자의 기대(이상적 인식) vs 저자가 경험한 현실" 간극을 보여주는 구조. 현재 As-Is(1-4부) → To-Be(5-6부) 서사와 정확히 일치
- **Fishbone(Ishikawa)** — 제3부(7-10장) 실패 분석에만 국소 적용. 실패 원인을 6개 범주(시간/인력/전문성/비용/시장/제도)로 구조화
- **적용 수준**: 저자의 내부 분석 원칙으로 사용. "Gap Analysis", "Fishbone" 등 용어는 본문에 미노출. 일상어로 녹여서 서술
- **불채택**: RCA/5Whys(선형 인과 분석이 복합적 구조 문제에 부적합), Causal Loop(학술적 인지 부담), FMEA(제6부 체크리스트가 이미 역할 수행)

## 핵심 제약 사항

1. **source/note.md는 READ ONLY** - 저자의 원본 메모. 절대 수정하지 않음
2. **IT 비유는 5장에만 집중** - 다른 장에 분산 배치하지 않음. 5장 이외에서는 "IT개발자/IT전문가" 표현 금지, "회사원/제도권 시스템 경험자"로 표현
3. **감성적/동기부여 표현 금지** - 냉정하고 건조한 톤 유지
4. **이모지 사용 금지**
5. **모든 내용은 한국어로 작성**
6. **5부(13장)는 축소 완료** - 4장/17절/66p → 1장/6절/15p. note.md 라인40-46의 6개 bullet point만 반영
7. **타입 에러 억제 금지** (코드 프로젝트가 아니지만 원칙으로 유지)
8. **비난/가치판단 표현 금지** - "책임 회피", "방관", "변질", "덫", "허상", "사기" 등 note.md에 없는 비난 표현 사용 금지. 구조적 현실 기술로 대체

## 저자 배경 정보

- 10여년 IT 개발자 및 관리자 경력
- 법인 설립 및 운영 경험
- 귀농 후 스마트팜 사업계획을 수립하여 후계농업경영인에 선정, 사업 진행
- 농업마이스터대학 수료 + 다수의 농업교육 이수
- 인삼(6년 재배 주기) 스마트팜 시도 후 실패
- 농업을 "5가지 전문 분야의 결합"으로 봄: 수학, 화학, 식물학, 토목공학, 회계학
- 스마트팜을 "농사의 자동화 3축"으로 정의: 환경제어, 영양분공급, 데이터수집
- 제도권 시스템에 익숙한 회사원의 논리적/분석적 관점이 책 전체에 녹아있음
- 본문에서는 IT개발자가 아닌 "평범한 회사원"으로 표현 (5장 IT비유 전담 제외)
